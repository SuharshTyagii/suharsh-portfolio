üè† House Robber Solution Explained

üß† The Big Picture

This problem is about making optimal decisions at each house. The key insight is that for any house, you have two choices:

  • Rob this house: You get its money + the max money from houses that are NOT adjacent
  • Skip this house: You get the max money from the previous house

The solution uses dynamic programming to keep track of the maximum money you can get up to each point.

üîç Understanding the Variables

rob1
Max money if we DON'T rob the previous house
rob2
Max money if we CAN rob the previous house

Think of these as "memory" - they remember the best decisions we've made so far!

üìù The Algorithm Step by Step

function rob(nums) { let rob1 = 0; // Max money 2 houses ago let rob2 = 0; // Max money 1 house ago for (const num of nums) { const temp = Math.max(num + rob1, rob2); rob1 = rob2; rob2 = temp; } return rob2; }

üí° Why This Works

At each house, we're asking: "What's the maximum money I can have at this point?"

We compare two scenarios:

  • Scenario 1: Rob current house + best result from 2 houses ago
  • Scenario 2: Don't rob current house, keep the best result from 1 house ago

We pick whichever gives us more money!

üéØ Key Insights

  • Space Efficient: We only need to remember the last 2 results, not all previous results
  • Time Efficient: We visit each house exactly once - O(n) time complexity
  • Greedy Won't Work: We can't just pick the highest values - we need to consider the constraint
  • Pattern Recognition: This is a classic "decision at each step" dynamic programming problem
